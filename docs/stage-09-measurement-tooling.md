# Stage 09: Measurement Tooling

Date: 2026-07-15

Status: complete for local parsing and runner integration

## Goal

Prepare the measurement layer needed for the final mobile feasibility decision.

The final answer cannot rely only on a generated video. It needs structured data:

- wall-clock latency
- peak GPU memory
- GPU utilization
- CPU resident memory
- output duration
- output resolution
- audio/video stream presence
- output file size
- run context: case id, seed, prompt hash, target duration/resolution, result path, and optional manifest hash

## Supported Inputs

### NVIDIA SMI CSV

Generated by:

```bash
nvidia-smi --query-gpu=timestamp,name,memory.used,memory.total,utilization.gpu --format=csv -l 1
```

Parsed fields:

- sample count
- GPU names
- peak memory used MiB
- total memory MiB
- average GPU utilization

### `/usr/bin/time -v`

Parsed fields:

- wall time seconds
- maximum resident set size
- user time
- system time
- exit status

### `ffprobe`

Parsed fields:

- duration seconds
- video stream count
- audio stream count
- width
- height
- format
- file size

### Run Context

Passed by `scripts/magihuman_task_runner.sh`:

- case id
- mode
- resolution
- model variant
- seed
- target duration seconds
- target base/SR dimensions
- result path
- prompt SHA-256
- optional manifest path and manifest SHA-256

## How To Use On GPU Host

After a run:

```bash
python -m backend.run_metrics \
  --log logs/<run>.log \
  --smi-csv logs/<run>_nvidia_smi.csv \
  --video outputs/smoke-test/<result>.mp4 \
  --case-id P01 \
  --mode t2v \
  --resolution 256p \
  --variant base \
  --seed 42 \
  --target-duration-seconds 5 \
  --target-br-width 448 \
  --target-br-height 256 \
  --result-path outputs/smoke-test/P01.mp4 \
  --manifest docs/p01-smoke-manifest.json \
  --output logs/<run>_metrics.json
```
