## 项目信息

项目名：MagiHuman Mobile Lab

GitHub 仓库名建议：`magihuman-mobile-lab`

项目介绍：

本项目用于复现和评估开源数字人生成模型 daVinci-MagiHuman。目标不是只跑通官方 demo，而是用可记录、可复查的数据判断它是否能被集成进手机 App。项目会依次完成官方资料核验、环境搭建、最小推理复现、性能测试、显存/耗时/模型体积记录、服务端接口验证，以及移动端部署可行性分析。

当前结论状态：

- [x] 根据文章和官方指标形成初步假设：完整模型大概率不适合手机端侧推理。
- [ ] 该结论还没有本地或云 GPU 实测数据支撑，不能作为最终结论。
- [ ] 最终结论必须基于以下实测数据：模型体积、峰值显存、推理耗时、依赖复杂度、输出质量、服务端封装成本、移动端接入成本。

## 判断标准

端侧 App 可行的定义：

- [ ] 模型和依赖能在 iOS / Android 本机运行。
- [ ] 不依赖远程 GPU 服务。
- [ ] 模型体积、内存占用、耗时、电量消耗在手机上可接受。
- [ ] 输出质量达到可演示标准。

云端 App 可行的定义：

- [ ] 模型可以在 GPU 服务器稳定推理。
- [ ] 手机端通过 API 上传输入并获取结果。
- [ ] 单任务耗时、排队、成本、失败重试机制可接受。
- [ ] 生成结果能被手机端顺畅播放、保存和分享。

最终结论必须写成三选一：

- [ ] A. 可以端侧部署。
- [ ] B. 不能端侧部署，但可以做“手机 App + 云端 GPU 推理”。
- [ ] C. 当前复现成本或效果不达标，暂不适合做产品化 App。
